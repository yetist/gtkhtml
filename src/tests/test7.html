eXtace 1.2.7
Dave J. Andruczyk - October 17th 1999, 06:52 EST

eXtace is a visual sound display/analysis program for the Gnome Desktop environment (though it works under other environments as long as ESD is installed). It requires ESD to function and includes various fourier transforms of the audio data in real-time. eXtace's displays include 3D textured flying landscape, 16-128 channel graphic EQ, scope, and a 3D pointed flying landscape. Listar 0.126a
Jeremy Blackman - October 16th 1999, 21:10 EST

Listar is a from-scratch implementation of a mailing list management software. It has a lot of similarities to Listserv which was one of it's inspirations, but has been written to support the features and abilities that the authors felt were lacking (or hard to modify/change) in some of the other list management programs available. It's main flexibility is that it is written around a plugin layer (similar to Apache) and this affords the ability to customize and extend the manager in ways that otherwise would be hard.

Changes: Support scripts have been eliminated (and creation of lists moved to the main executable), creation of lists now builds the config file dynamically from the installed modules and defaults, internal I/O changes have been made to allow Listar to handle large installations (e.g. 5000+ lists), internal qmail bounce detection support and the beginnings of support for DSN have been added, several file lock contention bugs have been fixed, and minor MIME handling updates have been made.

Urgency: medium

Dieresis Newsboy 1.0.0 Revision 99.10.16
Gerardo Betancourt - October 16th 1999, 21:05 EST

Newsboy is a tool for publishing simple news articles and announcements online. The script (programmed in Perl) can be used to publish news and announcements on a web site without the need to know HTML (or have Web development experience for that matter). The Newsboy script allows an administrator to create, modify and delete accounts for different users (or "editors"). The script features three levels of access to the news database. Level one is used for providing guest access to the database (viewing privileges only). Level two provides trusted news editors with the required privileges to add, edit or delete their own news postings. Level three is used for administrative purposes (adding, editing, and deleting news articles or categories, or adding, editing, and deleting editor accounts). The script was designed to enable a team of people to quickly post comments, reviews, or announcements with the option of providing hyperlinks without knowing HTML. The script can be considered a "gossiper," instead of a real news publisher.

Changes: The system protection subroutine has been modified to allow system administrators and/or webmasters to specify domain names other than the one used to run the script via a new variable located in the global configuration file.

Urgency: low

Drall 0.14.0
Henrik Edlund - October 16th 1999, 18:27 EST

Drall is a script which allows users to access their directories and files remotely without the need of using insecure ftp and telnet. It enables the user to treat the remote file system as if it was on their local hard disk trough a normal web browser. The interface resembles the well known Norton Commander (of DOS fame) and Midnight Commander (of UNIX fame). Drall relies on the server's HTTP authentication and SSL for security. Dual-frame interface makes it easy to overlook file system and the modular design means you only use the features you need. Written in Perl for easy customization and expansion.

Changes: Filter functionality. This is the feature freeze for the predicted stable release in late November.

Entity 0.3.8
Ian Main - October 16th 1999, 13:18 EST

Entity is an XML parser and renderer which can be used to build GTK GUI applications in XML. The XML can then be modified at runtime, and changes to the tree immediately reflect the onscreen representation. Using XML for the user interface, data storage, and code of an application allows you to easily implement configuration, saving, and undo because the state of the entire application is always reflected in the XML tree. This also makes modular design and plugins very easy to create and integrate. Entity currently supports Perl, Tcl, and C for doing runtime XML creation and modification.

Changes: Generalized language support, added Tcl and C, lots more supported tags, cleaned up core code and started on documentation.
